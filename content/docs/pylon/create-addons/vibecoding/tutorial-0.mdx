## 前言

所以，你不会写代码，但又想搞个 Rebar 附属实现你的脑洞？找对地方了！这套教程会手把手教你用 AI 来做 Rebar 附属。不过你还是要具备一些基本能力——如果你连电脑都不会开、鼠标都不会用，那跟这个教程可能会有点吃力。

开始之前先说几件事...

### 前置条件

我们假设：

- 你是个有经验的 Minecraft 玩家，知道 Rebar 大概是干啥的、能做什么。
- 你脑子里有个想法，想通过 Rebar 附属把它变成现实。
- 你愿意学习新东西（哪怕是用 AI 写代码也需要你动脑子）。

不需要会写代码！这就是 Vibecoding 的魅力所在。

### 什么是 Vibecoding？

Vibecoding 就是用 AI 来写代码的一种方式。简单来说：你告诉 AI 你想要什么，AI 帮你把代码写出来。对于 Rebar 附属开发来说，就是把"我想要一个能 XX 的物品/方块"这种自然语言描述，转换成实际的 Java/Kotlin 代码。

听起来很美好对吧？确实如此——但也不是没有坑。

### Vibecoding vs 传统写代码

Vibecoding 的优势很明显：

- **门槛低**：不需要懂编程语法、不需要理解设计模式
- **速度快**：想法到成品的时间大幅缩短
- **试错成本低**：改个需求就是一句话的事

但缺点也得心里有数：

- **代码质量看运气**：AI 生成的代码可能不如资深开发者写得优雅
- **隐藏 BUG**：有些问题运行时才会暴露
- **调试能力要求**：你得能看懂报错信息、能描述清楚问题出在哪
- **需要你参与**：AI 不是万能的，你需要不断反馈、调整、测试

<Callout type="info" title="核心原则">
  把 AI 当作你的**结对编程伙伴**，而不是魔法棒。你需要清晰地描述需求、验证输出结果、发现问题并反馈给 AI。Vibecoding = 你的创意 + AI 的编码能力 + 你的审核把关。
</Callout>

### 关于 Kotlin

虽然你可以让 AI 用纯 Java 写代码，但我们**强烈推荐 Kotlin**。原因很简单：

- **Token 消耗更少**：Kotlin 语法更简洁，同样的功能代码量更短
- **生成速度更快**：代码短意味着 AI 响应更快
- **Rebar 本身就用 Kotlin**：官方代码都是 Kotlin，AI 参考起来更准确
- **更好用的语法糖**：很多 Java 要写好几行的事，Kotlin 一行搞定

在本教程中，我们默认使用 Kotlin。

### "我卡住了，救命！"

1. **先自己试试**：很多时候问题比你想象的简单，再试一次说不定就通了。
2. **换个更强的模型**：如果当前模型搞不定，换 GLM 5.1 或 Claude Opus 4.6 试试。
3. **提供更多上下文**：把报错信息、相关代码片段都丢给 AI，别只说"它不工作"。
4. **拆分问题**：大问题拆成小问题，一步步解决。
5. **回退版本**：如果改坏了，回到上一个能工作的版本重新来过。

现在让我们开始吧！

## 环境搭建

### 选择你的 IDE 工具

Vibecoding 离不开两样东西：**AI 模型**和 **IDE 环境**。AI 服务通常要花钱（当然也有免费的），IDE 工具则决定了你的使用体验。

市面上主流的 AI 编程工具有不少，下面给你介绍几个常用的选择：

#### 1. Trae CN（国内版）

<Callout type="success" title="新手推荐">
  如果你在中国大陆、不想折腾网络、预算有限，Trae CN 是最省心的选择。
</Callout>

- **免费使用**，但要排队。高峰期可能要等一会儿才能用上，不过进去之后体验还不错。
- 集成在 IDE 里，适合不想离开编辑器的新手。
- **不需要科学上网**，国内网络直接访问。
- **不用自己准备模型**，打开就能用。
- 对电脑配置要求不高，普通开发机就能跑。

**适合人群**：想零成本尝鲜、不介意偶尔排队的新手玩家。

---

#### 2. Trae Solo 3.0（云端版）

- **对电脑配置没要求**：代码在云端跑，配置差的电脑甚至手机都能编程。
- 目前免费，起步成本为零。
- **缺点明显**：调试麻烦，环境受限——没法随便装系统依赖、连本地数据库，遇到奇怪的问题比较难排查，而且还得排队。
- 不需要科学上网，也不用自己准备模型。

**适合人群**：电脑性能一般、想随时随地用手机或旧电脑写代码的新手，但要接受环境限制。

---

#### 3. Claude Code

<Callout type="warning">
  上手门槛较高，新手慎入！
</Callout>

- **目前最强的 CLI 编程工具之一**，能力非常强，能自动完成复杂任务。
- **上手门槛很高**：需要熟悉命令行操作，还要**科学上网**（国内无法直接访问），并且**需要付费订阅**才能使用。
- 可以接入自己的模型，但配置过程也比较麻烦。

**适合人群**：有一定基础、不怕命令行、追求极致性能的进阶玩家。

---

#### 4. OpenCode

- **不需要科学上网**，国内直接用，这一点对新手很友好。
- **需要自己准备模型 API Key**：得自己去申请 OpenAI 或其他大模型的 Key，然后填进去。
- 开源免费，但"免费"的前提是你得搞定模型 Key（有些模型提供免费额度）。
- 也是命令行工具。

**适合人群**：动手能力强、愿意花时间配置 API、又想省钱的折腾型新手。

---

#### 5. OpenAI Codex

- **有免费套餐**，免费用户也能用，但额度有限。想多用可以买 Plus（$20/月）或 Pro（$100/月）。
- **不需要自己准备模型**，但是需要科学上网。
- 能力很强：不仅能写代码，还能跨应用操作电脑（比如打开浏览器、点击按钮）。

**适合人群**：想体验最新 AI 编程+电脑自动化、愿意每月花一点钱（或者先用免费额度）的新手，但需要解决网络问题。

### 网络配置

为了能从 GitHub 克隆项目模板，我们需要确保网络通畅。

<Callout type="info" title="为什么需要配网络？">
  Rebar 的模板仓库托管在 GitHub 上，国内直接访问可能会很慢甚至连不上。所以我们需要用一些工具来加速。
</Callout>

推荐使用免费的 **Watt Toolkit**（原名 Steam++）：

- [官网下载](https://steampp.net/download)
- [微软商店](https://apps.microsoft.com/detail/9mtcfhs560ng)

安装后怎么加速 GitHub？[这里有详细教程](https://zhuanlan.zhihu.com/p/1910668854859890688)

### 安装 Java

没有 Java 就像吃饭没带嘴——你没法构建 Rebar 附属。所以，先装好 Java。

推荐安装 **Zulu JDK 21**（LTS 版本，稳定可靠）：

1. 打开 [Azul 下载页面](https://www.azul.com/downloads/?version=java-21-lts&package=jdk#zulu)
2. 选择 **Windows MSI 安装包**（双击安装，最省心）
3. 一路"下一步"完成安装

装完后打开终端（PowerShell 或 CMD），输入 `java -version`，能看到版本号就说明装好了。

### 创建工作目录

打开你选择的 IDE（以 Trae CN 为例）：

1. 点击"打开文件夹"
2. 找个地方新建一个文件夹，比如叫 `MyRebarAddon`
3. **注意**：文件夹路径里**不要有空格**！这很重要，空格会导致各种奇怪的问题
4. 选中文件夹，打开

### 初始化项目

传统方式需要手动克隆仓库、创建分支、导入项目……但在 Vibecoding 里，这些全交给 AI！

直接对 AI 说：

```txt
我将要初始化环境，请你克隆 https://github.com/pylonmc/rebar-addon-template.git 到当前工作区，然后安装 Gradle 依赖，最后构建 JAR 文件。
```

<Callout type="success" title="最佳实践">
  给 AI 下指令时尽量**具体明确**。别说"帮我建个项目"，要说"克隆 XXX 仓库、安装依赖、构建 JAR"。指令越清晰，AI 执行得越准确。
</Callout>

等 AI 忙活完，项目应该就初始化好了。

### 运行测试服务器

附属组件模板自带 `run server` 功能，可以直接启动测试服务器。对 AI 说：

```txt
请在项目根目录下运行 ./gradlew runServer 来启动测试服务器。
```

![运行测试服务器](../img/running-test-server.png)

你会看到一个控制台窗口显示服务器日志。第一次运行大概需要一 two 分钟下载服务端文件。

初次运行通常会失败——因为你还没同意 EULA（最终用户许可协议）。对 AI 说：

```txt
服务器启动失败了，应该是没同意 EULA。请你进入 run 文件夹，打开 eula.txt 把 false 改成 true，然后重新运行 runServer 任务。
```

<Callout type="danger">
  关闭服务器时，**不要**点 IDE 的停止按钮！那样不会正常关闭服务器，可能导致进程残留。正确做法是在控制台输入 `stop`，或者在游戏内用 `/stop` 命令。
</Callout>

服务器成功启动后：

1. 用 Minecraft 客户端连接 `localhost:25565`
2. 在控制台输入 `op <你的用户名>` 给自己管理员权限
3. 试试获取默认的示例物品：`/rb give <用户名> exampleaddon:example_item`

如果能拿到示例物品，说明环境搭建成功了！接下来就可以开始真正的创作了。
