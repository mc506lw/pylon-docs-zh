Steamwork 是一个基于 **Pylon 与 Rebar** 的蒸汽与压力科技附属。它在 Pylon 已有的金属、流体、研究和指南体系之上，扩展出**锅炉产能、蒸汽流体输送和蒸汽驱动机器**的完整闭环。附属新增汽动物流网络、产线自动化、升级模组和精密加工，让蒸汽科技成为 Pylon 生态中的重要一环。

## 🎮 工作原理

### 锅炉链与蒸汽产出

Steamwork 的核心是**四级锅炉链**，每级锅炉产出不同品质的蒸汽：

```
青铜锅炉 (BronzeBoiler)     → 蒸汽 (Steam)
    ↓ 升级
因瓦锅炉 (InvarBoiler)       → 超热蒸汽 (Superheated Steam)
    ↓ 升级
锰钢锅炉 (ManganeseSteelBoiler) → 高压蒸汽 (High-Pressure Steam)
    ↓ 升级
钨锅炉 (TungstenBoiler)      → 高压蒸汽（最高效率）
```

所有锅炉继承自 `AbstractSteamBoiler`，共享以下核心机制：

- **水输入 → 蒸汽输出**：从下方输入水，向上方/侧面输出蒸汽
- **热源检测**：自动识别营火、熔岩、岩浆块等热源
- **启动惩罚**：冷启动时需要额外加热时间才能开始产汽
- **压力回退（Pressure Backoff）**：当缓冲区填充超过 50% 时，产汽速度逐渐降低，防止过载
- **泄漏机制**：超压时可能产生蒸汽泄漏
- **WAILA 显示**：实时显示压力状态（低 / 中 / 高 / 排放中）

### 蒸汽驱动的加工机器

所有加工机器继承自 `AbstractSteamProcessor`（约 1468 行的核心基类），实现了完整的配方生命周期：

```
findIngredients()  →  检查原料是否齐全
       ↓
consumeReserved()  →  预扣原料（预留）
       ↓
startRecipe()      →  开始配方，消耗蒸汽
       ↓
progressRecipe()   → 每tick推进进度，持续消耗蒸汽
       ↓
output             →  完成产出
```

机器状态枚举 `StopReason`：
| 状态 | 含义 |
| :--- | :--- |
| `NO_STRUCTURE` | 多方块结构不完整 |
| `READY` | 就绪，等待原料或蒸汽 |
| `NO_INGREDIENTS` | 缺少原料 |
| `NO_STEAM` | 缺少蒸汽供应 |
| `OUTPUT_FULL` | 输出槽已满 |
| `MIXED_OUTPUT` | 输出类型冲突 |
| `PROCESSING` | 正在处理中 |

### 涡轮加速机制

涡轮（`AbstractSteamBooster` 子类）可以为其半径内的机器提供**加速 Boost**：

- **基础蒸汽涡轮（SimpleSteamTurbine）**：消耗蒸汽，加速附近的 Steamwork 机器
- **精密蒸汽涡轮（PrecisionProcessingTurbine / PrecisionSteamTurbine）**：更高效率的加速
- **液压/柴油涡轮（HydraulicTurbine / DieselTurbine）**：使用高级流体的强力涡轮

**Boost 规则：**
- 全局上限 `MAX_BOOST_STACKS = 3`，每个方块每 tick 最多叠加 3 次 boost
- 支持的目标类型：`VANILLA_FURNACE`（原版熔炉）、`STEAMWORK_BOOSTABLE`（Steamwork 可加速机器）、`REBAR_PROCESSOR`（Rebar 处理器）、`REBAR_RECIPE_PROCESSOR`（Rebar 配方处理器）
- 扫描半径可通过 **RANGE 升级模组**扩展
- 支持**协同乘数**——受影响的机器越多，整体效率越高

### 汽动输送网络

Steamwork 提供了一套类似 CargoDuct 风格的**汽动物流网络**：

```
输出口 (PneumaticOutput)  ←── 管道网络 (PneumaticDuct) ──→  输入口 (PneumaticInput)
                              ↕
                         阀门 (PneumaticGateValve)
                              ↕
                       分配器 (PneumaticDistributor)
                              ↕
                     货物枢纽 (PneumaticCargoHub)
```

- **管道（PneumaticDuct）**：使用 ItemDisplay 渲染的视觉管道，最多 3 个连接面，支持 Z-fighting 防止
- **BFS 寻路**：`findReachableEndpoints()` 从起点广度优先搜索所有可达端点
- **双向连接验证**：管道之间需要互相确认连接关系（reciprocal connection）
- **Facade 支持**：可覆盖外观方块隐藏管道

### 产线系统

通过**蓝图道具（ProductionLineBlueprint）** 将多台机器串联成直线流水线：

```
产线入口 (Inlet) → 机器1 → 机器2 → ... → 机器N → 产线出口 (Outlet)
```

**关键特性：**
- 支持接入 **Steamwork 加工机器**、**Pylon 机器**、**原版熔炉/工作台**
- 物品通过 `deliverToNextMember()` 自动向下游推送
- 产线信息通过 **PDC（PersistentDataContainer）** 持久化存储，服务器重载后可恢复
- 支持多方块机器的附属方块跳过（最大间隔 `DISBAND_MAX_GAP = 3` 格）
- 入口可安装 **AUTO_PRODUCTION 升级模组** 自动触发需手动操作的机器
