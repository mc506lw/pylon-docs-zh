## 🧑‍💻 快速开始

RebarWrench 为附属开发者提供了一套简洁的 API，让你的 Rebar 方块支持扳手交互。

### 添加依赖

将 RebarWrench 添加到你的 `build.gradle.kts` 中：

```kotlin
dependencies {
    compileOnly("io.github.lijinhong11:RebarWrench:1.0.2")
}
```

也可以通过 [Maven Central](https://central.sonatype.com/artifact/io.github.lijinhong11/RebarWrench) 获取最新版本。

### 基本集成

要让一个 Rebar 方块支持扳手，只需通过 `Wrenchable.builder()` 构建一个 `Wrenchable` 实例，并在加载时注册：

```java title="MyMachine.java"
import io.github.lijinhong11.rebarwrench.RebarWrench;
import io.github.lijinhong11.rebarwrench.api.WrenchAction;
import io.github.lijinhong11.rebarwrench.api.WrenchResult;
import io.github.lijinhong11.rebarwrench.api.Wrenchable;
import io.github.pylonmc.rebar.block.RebarBlock;
import io.github.pylonmc.rebar.block.base.RebarInteractBlock;
import io.github.pylonmc.rebar.block.context.BlockCreateContext;
import net.kyori.adventure.text.Component;
import net.kyori.adventure.text.format.NamedTextColor;
import org.bukkit.NamespacedKey;
import org.bukkit.block.Block;
import org.bukkit.entity.Player;
import org.bukkit.event.EventPriority;
import org.bukkit.event.player.PlayerInteractEvent;
import org.bukkit.persistence.PersistentDataContainer;
import org.bukkit.persistence.PersistentDataType;
import org.jspecify.annotations.NonNull;

public class MyMachine extends RebarBlock implements RebarInteractBlock {
    private final NamespacedKey INT = new NamespacedKey(PLUGIN_INSTANCE, "ivalue");
    private int colorIndex = 0;
    private int value = 0;
    private final NamedTextColor[] COLORS = {NamedTextColor.BLUE, NamedTextColor.GREEN};

    public static final Wrenchable WRENCHABLE = Wrenchable.builder()
            .interactFunction((Player player, RebarBlock block, WrenchAction action) -> {
                MyMachine machine = (MyMachine) block;
                return switch (action) {
                    case CONFIGURE -> {
                        // 右键：切换颜色
                        machine.colorIndex = (machine.colorIndex + 1) % machine.COLORS.length;
                        player.sendActionBar(Component.text("Color: " + machine.COLORS[machine.colorIndex].toString()));
                        yield WrenchResult.SUCCESS;
                    }
                    case ROTATE -> {
                        // Shift + 右键：旋转数值
                        machine.value = (machine.value + 1) % 6;
                        player.sendActionBar(Component.text("Value: " + machine.value));
                        yield WrenchResult.SUCCESS;
                    }
                    case FAST_BREAK -> {
                        // Shift + 左键：允许快速拆解
                        yield WrenchResult.SUCCESS;
                    }
                };
            })
            .build();

    public MyMachine(@NonNull Block block, @NonNull BlockCreateContext context) {
        super(block, context);
    }

    public MyMachine(@NonNull Block block, @NonNull PersistentDataContainer pdc) {
        super(block, pdc);
    }

    @Override
    public void onInteract(@NonNull PlayerInteractEvent e, @NonNull EventPriority priority) {
        if (!e.getAction().isRightClick()) return;
        e.getPlayer().sendMessage("The int value is " + value);
    }

    @Override
    public void write(@NonNull PersistentDataContainer pdc) {
        pdc.set(INT, PersistentDataType.INTEGER, value);
    }

    // 在你的附属加载时注册
    public static void register() {
        RebarWrench.registerWrenchable(MyMachine.class, WRENCHABLE);
    }
}
```

## 🔧 API 参考

### Wrenchable

核心接口，定义方块被扳手交互时的行为。

```java title="Wrenchable.java"
public interface Wrenchable {
    WrenchResult onWrenchInteract(Player player, RebarBlock block, WrenchAction action);

    static Builder builder() {
        return new WrenchBuilder();
    }
}
```

| 方法 | 返回值 | 说明 |
| :--- | :--- | :--- |
| `onWrenchInteract(Player, RebarBlock, WrenchAction)` | `WrenchResult` | 处理扳手交互，根据动作类型执行对应逻辑并返回结果 |
| `builder()` | `Builder` | 获取构造器实例 |

### WrenchAction

操作类型枚举，表示玩家使用扳手触发的具体操作。

```java title="WrenchAction.java"
public enum WrenchAction {
    FAST_BREAK,   // Shift + 左键：快速拆解
    CONFIGURE,    // 右键：配置
    ROTATE        // Shift + 右键：旋转
}
```

| 枚举值 | 触发方式 | 典型用途 |
| :--- | :--- | :--- |
| `CONFIGURE` | 右键点击方块 | 切换配置选项（如颜色、模式） |
| `ROTATE` | 按住 Shift + 右键点击方块 | 旋转或循环数值（如朝向、档位） |
| `FAST_BREAK` | 按住 Shift + 左键点击方块 | 快速拆解方块（需在配置中启用） |

### WrenchResult

操作结果枚举，告知扳手本次交互的处理结果。

```java title="WrenchResult.java"
public enum WrenchResult {
    SUCCESS,     // 操作成功执行
    FAILED,      // 操作失败（玩家会收到"此操作不支持"提示）
    UNSUPPORTED  // 该操作不被支持（等同于默认行为）
}
```

| 枚举值 | 说明 |
| :--- | :--- |
| `SUCCESS` | 操作成功完成。对于 `FAST_BREAK`，返回此值将触发方块破坏 |
| `FAILED` | 操作失败，玩家会收到错误提示消息 |
| `UNSUPPORTED` | 不支持该操作（默认值），无特殊效果 |

### Wrenchable.Builder

使用构造器模式构建 `Wrenchable` 实例。

| 方法 | 说明 |
| :--- | :--- |
| `interactFunction(InteractHandler function)` | 设置交互回调函数，接收 `(Player, RebarBlock, WrenchAction)` 并返回 `WrenchResult` |
| `build()` | 创建不可变的 `Wrenchable` 实例 |

#### InteractHandler 函数签名

```java
// InteractHandler 是一个函数式接口
WrenchResult handle(Player player, RebarBlock block, WrenchAction action);
```

推荐使用 Java 的 switch 表达式来处理不同的 `WrenchAction`：

```java
.interactFunction((player, block, action) -> {
    return switch (action) {
        case CONFIGURE -> { /* 处理配置逻辑 */ yield WrenchResult.SUCCESS; }
        case ROTATE -> { /* 处理旋转逻辑 */ yield WrenchResult.SUCCESS; }
        case FAST_BREAK -> WrenchResult.SUCCESS; // 或 WrenchResult.FAILED 以拒绝快速拆解
    };
})
```

### 注册与获取

在你的附属启用阶段注册 `Wrenchable`：

```java
@Override
public void onEnable() {
    MyMachine.register();
}
```

你也可以在运行时获取已注册的 `Wrenchable`：

```java
Wrenchable wrenchable = RebarWrench.getWrenchable(MyMachine.class);
```

| 方法 | 说明 |
| :--- | :--- |
| `RebarWrench.registerWrenchable(Class<? extends RebarBlock>, Wrenchable)` | 为指定的 RebarBlock 类注册扳手交互行为 |
| `RebarWrench.getWrenchable(Class<? extends RebarBlock>)` | 获取指定类已注册的 `Wrenchable` 实例（未注册则返回 `null`） |

## 📝 最佳实践

- 在 `interactFunction` 中使用 switch 表达式处理所有三种 `WrenchAction`，确保每种情况都有明确的返回值
- 对于不应被快速拆解的核心机器方块，在 `FAST_BREAK` 分支中返回 `WrenchResult.FAILED`
- 使用 `player.sendActionBar()` 向玩家反馈操作结果，提供良好的用户体验
- 所有状态修改应在回调中直接完成，保持逻辑集中
- 避免在 `interactFunction` 中执行耗时操作，因为它运行在主线程上
- 如果某个操作在当前状态下无意义，返回 `WrenchResult.FAILED` 会向玩家显示提示信息
