# 方块接口

Rebar 有许多内置接口，可以为方块添加自定义行为。例如，有一些接口允许您：

- 添加流体缓冲区（[RebarFluidBufferBlock](https://pylonmc.github.io/rebar/docs/javadoc/io/github/pylonmc/rebar/block/base/RebarFluidBufferBlock.html)）
- 在点击方块时运行代码（[RebarInteractBlock](https://pylonmc.github.io/rebar/docs/javadoc/io/github/pylonmc/rebar/block/base/RebarInteractBlock.html)）
- 当玩家跳到方块顶部时运行代码（[RebarJumpBlock](https://pylonmc.github.io/rebar/docs/javadoc/io/github/pylonmc/rebar/block/base/RebarJumpBlock.html)）
- ...还有很多很多

!!! question "如何使用接口 X？"
    1. 检查此站点上是否有任何文档（向下滚动）。
    2. 阅读接口的 JavaDocs 和相关方法的文档。
    3. 在 Pylon 或其他附加组件中查找使用该接口的示例。
    4. 在 Pylon Discord 上寻求帮助。

例如，假设您想在右键点击方块时生成粒子。您可以使用 [RebarInteractBlock](https://pylonmc.github.io/rebar/docs/javadoc/io/github/pylonmc/rebar/block/base/RebarInteractBlock.html) 实现：
```java title="ExampleBlock.java"
public class ExampleBlock extends RebarBlock implements RebarInteractBlock {

    // ...

    @Override
    public void onInteract(@NotNull PlayerInteractEvent event, @NotNull EventPriority priority) {
        if (!event.getAction().isRightClick()
                || event.getPlayer().isSneaking()
                || event.getHand() != EquipmentSlot.HAND
        ) {
            return;
        }

        new ParticleBuilder(Particle.CAMPFIRE_COSY_SMOKE)
                .location(getBlock().getLocation().toCenterLocation().add(0, 0.7, 0))
                .count(smokeCount)
                .extra(0.1) // speed
                .spawn();
    }
}
```

## `postLoad` 方法

许多方块接口存储持久数据。而不是强迫您在自己的 `write` 方法和加载构造函数中自行加载和保存数据，Rebar 的所有方块接口都有一个单独的机制，用于将数据保存到方块的 PDC。这样做的一个副作用是，在加载构造函数被调用时，Rebar 的方块接口都还没有加载它们的数据。因此，作为经验法则，您不应该在加载构造函数中调用与方块接口相关的任何方法。相反，覆盖 `postLoad` 方法并在那里执行您需要的操作。`postLoad` 方法将在与块关联的所有方块接口加载它们的数据后被调用。

!!! info "postInitialise"
    还要注意 `postInitialise` 方法，它与 `postLoad` 相同，只是它也在 place 构造函数之后被调用。
