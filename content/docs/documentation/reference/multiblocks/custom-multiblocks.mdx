---
title: 自定义多方块
description: 了解如何使用 RebarMultiblock 创建自定义多方块
---

`RebarSimpleMultiblock` 非常有用且适用于大多数多方块。然而，有时您可能需要更大的灵活性。一个简单的例子是流体 tanks；一个多方块，它可以在其顶部有 0-8 个多方块储罐外壳。显然，这不适用于常规的 `RebarSimpleMultiblock`。

这可以使用 `RebarMultiblock` 来完成，`RebarSimpleMultiblock` 就是构建在其之上的。在它的核心，多方块系统只是一个变更检测系统，它监听方块被放置或破坏的情况，并在您的多方块有组件放置或破坏时通知它。为了做到这一点，变更检测系统需要您提供：

* 多方块跨越的区块列表
* 一个函数，用于返回给定方块是否是多方块的一部分

要提供这些，您的多方块必须实现 `getChunksOccupied` 和 `isPartOfMultiblock`。例如：

```java
@Override
public @NotNull Set<@NotNull ChunkPosition> getChunksOccupied() {
    return Set.of(new ChunkPosition(getBlock().getChunk()));
}

@Override
public boolean isPartOfMultiblock(@NotNull Block otherBlock) {
    return otherBlock.equals(getBlock().getRelative(BlockFace.DOWN));
}
```

注意，`isPartOfMultiblock` 如果方块**可能**是多方块的一部分，则必须返回 true，因此您不应该进行任何针对方块*类型*的检查 - 只是位置！例如，如果您的多方块需要在它下方的方块有一个火，那么 `isPartOfMultiblock` 应该始终在向它提供方块下方的方块时返回 true。

此外，您必须实现 `checkFormed`，例如：

```java
@Override
public boolean checkFormed() {
    Block belowBlock = getBlock().getRelative(BlockFace.DOWN);
    return !BlockStorage.isRebarBlock(belowBlock) && belowBlock.getType() == Material.FIRE;
}
```

这将在与您的多方块（即在 `getChunksOccupied` 提供的区块中并且在 `isPartOfMultiblock` 中满足条件的方块）的任何方块被修改时被调用。它可以被看作是一个"刷新"函数，用于确保多方块仍然正确构造。

`RebarMultiblock` 仍然带有 `isFormedAndFullyLoaded`、`onMultiblockFormed` 和 `onMultiblockUnformed` 方法，这些方法对于实现您自己的多方块管理逻辑很有用。

显然，这是非常 barebones 的。您需要自己实现简单概念如旋转 - 您可能希望查看 `RebarSimpleMultiblock` 的源代码来使这更容易。

最后，如果您想给您的自定义多方块像 `RebarSimpleMultiblock` 那样的虚影方块，您可以使用 [RebarGhostBlockHolder](https://pylonmc.github.io/rebar/docs/javadoc/io/github/pylonmc/rebar/block/base/RebarGhostBlockHolder.html)。这会自动管理虚影方块。您可以使用 `addGhostBlock` 创建新的虚影方块，使用 `removeGhostBlock` 移除虚影方块。
