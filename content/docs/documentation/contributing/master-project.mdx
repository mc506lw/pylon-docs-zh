# 并行开发仓库项目

PylonMC 提供了一个叫 `parallel-dev-repo` 的工具仓库。它让你可以用自己修改过的核心代码来运行附属模组，测试新功能时非常方便。入门指南里用的就是这个方案。我们建议你用这个仓库同时修改 Pylon 和 Rebar，后面的内容也默认你正在使用它。

## 项目结构
```
pylon/
  src/
rebar/
  dokka-plugin/
  nms/
  rebar/
  test/
```

`pylon` 是基于 Rebar 的附属模组。`rebar` 下有四个子项目：

- `dokka-plugin`：自定义 Dokka 插件，用于优化 Javadocs 的格式化输出
- `nms`：所有涉及服务端内部（NMS）的 Rebar 代码，单独放在一个子项目中以便隔离不稳定的 API
- `rebar`：Rebar 的主体代码
- `test`：Rebar 的集成测试

## 任务

Pylon 主项目提供了一些常用的 Gradle 任务：

| 任务 | 别名 | 描述 |
|------|------|------|
| `runServer` | `runSnapshotServer` | 使用当前版本的 Rebar 和 Pylon 启动 Minecraft 服务器 |
| `:pylon:runServer` | `runStableServer` | 使用最新稳定版 Rebar 和当前版本 Pylon 启动服务器 |
| `:rebar:test:runServer` | `runLiveTests` | 运行 Rebar 集成测试 |

<Callout type="error">
  在 IntelliJ 中使用任务别名并附加调试器时会失效。解决方案：不要用别名，直接运行原始任务名。比如别用 `runSnapshotServer`，改用 `runServer`。
</Callout>
