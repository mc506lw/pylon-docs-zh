本指南将引导您完成为本文档站点做贡献的流程。该站点由 [MkDocs](https://www.mkdocs.org/) 和 [GitHub Pages](https://pages.github.com/) 提供支持。

## 第0步：前置条件

在开始之前，请确保您的系统上已安装以下内容：

### Git

Git 是克隆仓库和管理更改所必需的。

您可以使用 [Git](https://git-scm.com/install/) 命令行界面，或者使用 GitHub Desktop 等图形界面工具（推荐）。

### GitHub 账号

您需要一个 GitHub 账号来 fork 仓库并创建拉取请求。

如果您还没有账号，可以[免费创建一个](https://github.com/signup)。

### Python 和 pip

MkDocs 需要在您的系统上安装较新版本的 [Python](https://www.python.org/) 及其包管理器 [pip](https://pip.pypa.io/en/stable/installation/)。

您可以从命令行检查是否已安装这些软件（版本可能不同）：

```bash
$ python3 --version
Python 3.13.3
$ pip3 --version
pip 25.2
```

如果未安装 Python，您可以从[官方 Python 下载页面](https://www.python.org/downloads/)下载。

!!! note "Windows 用户注意"
    安装过程中，如果安装程序提供"Add Python to PATH"选项，请确保勾选（默认通常是关闭的）。

    ![Add Python to PATH](img/python-windows-path.png)

## 第1步：Fork 仓库

现在访问[文档仓库](https://github.com/pylonmc/docs)。
点击页面右上角的"Fork"按钮，创建您自己的仓库副本。

![GitHub Fork](img/github-fork.png)

---

## 第2步：克隆您的 fork

您需要将 fork 的仓库克隆到本地计算机，以便预览更改。

如果您使用 GitHub Desktop，请按照[这些步骤](https://docs.github.com/en/desktop/adding-and-cloning-repositories/cloning-a-repository-from-github-to-github-desktop)操作。

如果使用命令行，请前往 GitHub 上的 fork 仓库，点击绿色的"Code"按钮复制 HTTPS URL。

![GitHub Clone URL](img/github-clone.png)

然后执行以下命令：

```bash
git clone <REPLACE_URL_HERE>
```

---

## 第3步：安装依赖

现在您的本地已经有了仓库，接下来需要安装所需的依赖。

在 `docs` 目录中运行：

```bash
pip install -r requirements.txt
```

这将安装 MkDocs 和所有其他必要的软件包。

---

## 第4步：进行更改

现在您可以开始编辑文档文件了！

!!! warning
    您对 fork 所做的所有更改在通过拉取请求提交之前，都不会影响官方文档。

文档文件使用 Markdown 编写。如果您需要语法帮助，可以查阅[Mastering Markdown](https://guides.github.com/features/mastering-markdown/)或查看[此页面的 Markdown 文件](https://github.com/pylonmc/docs/blob/main/docs/en/documentation/contributing/contributing-to-docs.md?plain=1)。

### 4.1 了解文档结构

文档按语言组织在 `docs/` 目录中：

```text
docs/
├── en/          # 英文文档
├── zh-CN/       # 简体中文文档
└── ...          # 其他语言
```

对于英文文档，所有页面都位于 `docs/en/` 中。内容按不同章节和子目录组织。

!!! danger "重要"
    目前我们只接受英文文档的贡献，因为 Rebar 和 Pylon 仍在进行重大更改。请关注未来关于其他语言贡献的更新。

每个章节都有自己的目录，该章节的图像应放在同一目录下的 `img/` 文件夹中。

### 4.2 创建新页面

在 `docs` 目录中打开代码编辑器，按照以下步骤操作：

1. 导航到 `docs/en/` 中的适当子目录（例如 `docs/en/installation/`）
2. 创建一个具有描述性名称的 `.md` 文件

**文件命名规则：**

- 文件名必须以 `.md` 结尾
- 使用小写字母
- 使用连字符 `-` 代替空格
- 不要使用特殊字符
- 示例：`installing-addons.md`、`custom-recipes.md`、`getting-started.md`

然后：

1. 使用 [Markdown 语法](https://guides.github.com/features/mastering-markdown/)编写内容
2. 保存文件
3. 预览更改（参见[第5步](#step-5-previewing-your-changes)）

### 4.3 编辑现有页面

1. 在代码编辑器中打开要编辑的文件
2. 进行更改
3. 保存文件
4. 预览更改（参见[第5步](#step-5-previewing-your-changes)）

### 4.4 添加图片

#### 图片命名约定

- 仅使用小写字母
- 使用连字符 `-` 代替空格
- 使用描述性名称
- 尽可能使用 PNG 格式
- 示例：`github-fork.png`、`custom-item-example.png`、`crafting-table.png`

#### 图片放置位置

图片应放在与 markdown 文件**同一目录**的 `img/` 文件夹中。

例如：

- 如果您的页面是 `docs/en/installation/installing-pylon.md`
- 将图片放在 `docs/en/installation/img/` 中

如果该目录中不存在 `img/` 文件夹，请先创建它。

#### 在页面中使用图片

要在 markdown 文件中嵌入图片，请使用以下语法：

```markdown
![Image description](img/your-image-name.png)
```

**示例：**

```markdown
![GitHub Fork button](img/github-fork.png)
```

## 第5步：预览更改

在提交更改之前，最好在本地预览它们以确保一切看起来都正确。

在 `docs` 目录中运行以下命令：

```bash
mkdocs serve --livereload
```

这将启动一个本地 Web 服务器。在浏览器中打开 [`http://127.0.0.1:8000`](http://127.0.0.1:8000) 查看带有您更改的文档站点。

预览会在您保存文件更改时自动更新。

---

## 第6步：提交和推送更改

当您对更改满意后，需要将它们提交并推送到您的 fork。

如果您使用 GitHub Desktop，请按照[这些步骤](https://docs.github.com/en/desktop/making-changes-in-a-branch/committing-and-reviewing-changes-to-your-project-in-github-desktop)操作。

如果使用命令行，请在 `docs` 目录中运行以下命令：

```bash
git add .
git commit -m "描述更改的提交信息"
git push
```

!!! tip
    编写清晰且描述性的提交信息，解释您做了什么以及为什么这样做。

---

## 第7步：创建拉取请求

将更改推送到 GitHub 上的 fork 后，您可以创建拉取请求来提交更改。

1. 前往 GitHub 上的 fork 仓库。您始终可以在 GitHub 仪表板或您的个人资料页面上找到您的 fork。
2. 点击"Pull requests"选项卡。
   ![GitHub Pull Requests Tab](img/github-pull-requests.png)
3. 点击绿色的"New pull request"按钮。
4. 填写拉取请求的标题和描述。
5. 点击"Create pull request"。

就这样！您的更改将经过维护者审查。如果一切正常，您的贡献将被接受。

---

## 第8步：修改您的拉取请求

如果您已经提交了拉取请求但需要做出更改（例如，审查者请求更改），您可以轻松完成。

只需重复[第4步](#step-4-making-your-changes)到[第6步](#step-6-committing-and-pushing-your-changes)来制作和提交您的新更改。

拉取请求将自动更新以包含您的新更改，直到它被合并或关闭。
