# 开始指南

Rebar 使用 [Kotlin](https://kotlinlang.org/) 编写，这是一种与 Java 类似但具有更现代特性和简洁语法的语言。如果您了解 Java，您将能够很快掌握 Kotlin。

Pylon 使用 Java 编写。

## 如何开始

1. 克隆 `parallel-dev-repo` 仓库：`git clone https://github.com/pylonmc/parallel-dev-repo`（或使用 GitHub Desktop 等图形界面工具）
2. 如果您使用 IntelliJ，它将自动设置所有内容。如果没有，请运行 `./gradlew`。这将克隆 `rebar` 和 `pylon` 仓库。
3. 如果您想将更改提交到主项目，**删除 `rebar` 或 `pylon` 目录（取决于您想向哪个项目贡献），fork rebar 或 pylon 仓库，并将您的 fork 克隆到同一目录中。** 否则，您将无法为更改打开拉取请求（除非您是 Rebar/Pylon 开发人员并且有权访问 Rebar/Pylon 仓库）。

有关主仓库的更多信息，请参阅[并行开发仓库项目](master-project.md)页面。

## 提交您的贡献

我们通常欢迎对 Rebar 和 Pylon 的贡献，但在进行任何重大更改之前最好先与 Pylon 团队核实，因为我们已经计划了一些可能不适合您更改的内容。如果您有兴趣做任何重大工作，或对贡献有任何疑问，请加入我们的 Discord 服务器与我们聊聊天 :)

完成更改后，打开一个拉取请求并提供有关您做了什么以及为什么这样做的信息。

## 测试

Rebar 有一套集成测试。应该仅为关键功能（如方块存储和配方）添加测试。

## 自定义 Dokka

Pylon 使用自定义版本的 Dokka 来生成 Javadocs。这是因为我默认的 Dokka 输出非常有问题且看起来不太好。Seggan 已向 Dokka 项目提交了修复这些问题的拉取请求，但尚未合并，因此在此期间我们使用自定义版本。如果您希望查看"修复后"的文档输出，请按以下步骤操作：

1. 克隆 `pylonmc/dokka` 仓库：`git clone https://github.com/pylonmc/dokka`
2. 检出 `pylon` 分支：`git checkout pylon`
3. 在根目录执行 `./gradlew publishToMavenLocal -Pversion=2.1.0-pylon-SNAPSHOT`。请注意，由于 Dokka 是一个非常庞大的项目，首次构建需要很长时间。在 Seggan 的相当不错的笔记本电脑上，首次运行大约需要 10 分钟。只要不删除构建缓存，后续构建将快得多。
4. 现在，在 Pylon 主项目中，执行 `./gradlew :rebar:rebar:dokkaGenerate -PusePylonDokka=true`。生成的输出位于 `pylon/rebar/rebar/build/dokka`。

## 我卡住了，该怎么办？

1. 如果是 Pylon 特有的问题，检查是否在文档中。如果不是 Pylon 特有的，就用谷歌搜索。
2. 在相关仓库中搜索问题，看看是否有人提到过
3. 在我们的 Discord 服务器上搜索相关术语，看看之前是否讨论过
4. 在我们的 Discord 服务器上提问
