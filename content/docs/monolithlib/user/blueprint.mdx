# 蓝图使用指南

蓝图是 MonolithLib 的核心概念，它定义了多方块机器的结构模板。开发者创建好蓝图后，玩家可以通过命令获取并使用。

## 获取蓝图

使用 `/ml blueprint` 命令获取蓝图物品：

```
/ml blueprint <ID>
```

获取的蓝图物品可以放在物品栏中，右键点击地面即可放置。

## 预览结构

在放置蓝图之前，可以先预览结构：

```
/ml preview
```

对准 Rebar 控制器使用，会显示投影方块，告诉你如何搭建。

## 放置建造

获取蓝图物品后，右键点击地面即可开始建造。

### 放置步骤

1. 使用 `/ml blueprint <ID>` 获取蓝图物品
2. 手持蓝图物品，右键点击地面放置
3. 对准控制器使用 `/ml build` 开始建造
4. 当所有材质匹配后，系统会自动修正方块状态

### 材质匹配

MonolithLib 的高级验证引擎只检查材质匹配，忽略方块状态：

- ✅ 任意朝向的楼梯都视为有效
- ✅ 任意类型的半砖都视为有效
- ✅ 其他方块状态差异也会被忽略

### 状态修正

当所有材质匹配完成后：
- 系统自动将所有方块状态修正为 Rebar 要求的精确状态
- 楼梯会自动调整到正确朝向
- 半砖会自动设置为正确类型

## 建造模式

### 轻松放置模式

```
/ml litematica easybuild
```

开启后，右键自动放置方块，适合快速建造。

### 自动打印模式

```
/ml litematica printer
```

开启自动打印功能。

## 常见问题

### Q: 为什么我的蓝图放置后没有反应？

A: 请确保所有方块的材质都正确匹配。MonolithLib 会先进行材质验证，只有全部匹配后才会触发状态修正。

### Q: 预览模式会影响服务器性能吗？

A: 不会。预览模式仅渲染玩家周围 7 格的投影方块，对服务器性能几乎没有影响。

### Q: 什么是双阶段蓝图？

A: 双阶段蓝图包含脚手架阶段和成型阶段。脚手架阶段用于玩家放置方块（宽松验证），成型阶段自动修正状态并绑定 Rebar 逻辑。
